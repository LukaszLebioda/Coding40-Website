Steps to create and manage a database...

...located on the external server far far away

Step 1 - Learn SQL

It's a language to construct sql queries, kind of sentences, to manage the content of databases located both on Your local system as well as on the external server.

Don't be frightened though. The language may look like egyptian hieroglyphs at first glance but it becomes as clear as the morning sky with just a little bit of practice.

Step 2 - Play with RDBMS

RDBMS stands for 'Relative Database Management System' and means a server-side system to host relative databases.

The most popular relative systems are nowadays MySQL and PostgreSQL. The latter is more complex, so I'd recommend starting with MySQL. There are some non-relative systems as well, e.g. MongoDB, that don't use tables and relations between the pieces of data. For now we will skip them however.

Step 3 - Turn Your local machine into a server

There is a mysteriously named package - XAMPP - that provides an Apache server, MySQL database and PHP interpeter (and more).

It's powerfull, open source and it allows Your PC to disguise as a server. Get familiar with all the new possibilities and consequences of that situation in safe environment before You jump into a real server jungle out there.

Step 4 - Build Your first database

Use phpMyAdmin - a web-based application with a user-friendly GUI to manage databases. It comes with XAMPP, with MySQL Workbench, it's omnipresent actually.

Databases are logical. They consist of tables, and tables consist of columns and rows, just like Excell. Each row is a record containing values (e.g. name, surname, age, sex, nationality - if You build a database of Your followers). There is a special number value however, a unique ID called 'primary key', to unmistakable recognize each record.

The relations between pieces of data located in separate tables gave the name to the whole system.

Step 5 - Build Your own webpage

That step takes a hell of a time to accomplish actually. The knowledge of html, css and javascript is a must-have which makes it one of the hardest.

But if You endure and prevail all the difficulties, Your own website will open the portal to the other universe for You. Reserve a domain for it, get the hosting service up and running and a ssl certificate to secure the Internet connection.

Now You're ready to make another step. Cause Your newborn website will give Your Visitors the opportunity to enter some data to the database You created.

Step 6 - Node.js

Some programming language is required to establish the API (Application Programming Interface): an interface between the User and the database.

When somebody visits Your page and fill any kind of form You’d prepared, the API should grab the input (called the request) and transfer it to the database to get / update / verify some data. The same API should then transfer the data retrieved from the database (called the response this time) back to Your page and displays it to the User.

You can use JavaScript to built Your API, as it is no longer front-end language only and was finally released from the browser’s custody to a multiplatform runtime for server-side applications, called node.js. So if You're a front-end developer intending to gain some server-side experience, You don't have to learn PHP, Ruby or Python!

Step 7 - NPM

There are countless packages (modules, dependencies) the programmers can import to their projects thanks to the courtesy of NPM (node package manager) to expand the possibilities of node.js.

To name just a few of those packages: express (a framework to build web pages / web applications that helps to handle HTTP responses & requests, sessions, routings etc.), mysql2 (the driver that allows to access aMySQL database), dotenv (imports environmental variables from .env files to store credentials and db access data safely), nodemon (a helpful tool that automatically restarts any node application You’re working on when file change is detected).

Feel free to create Your own NPM package and store it in the NPM repository like this.
