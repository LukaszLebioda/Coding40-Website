User story, ep. 1: Enlightment

Lukasz has recently discovered the diverse world of IT which had always been there
but Lukasz was too blind to see it.

The more blind he had been, the more intense his experiencing of IT-environment is nowadays.

After having tasted different flavours of IT he finally realized that all his experiences,
all his actions and all the decisions he made led to this precious moment:

to the resolution of becoming a software tester...

User story, ep. 2: Theory

So he started where many university-graduates think they should start from - buying / borrowing / downloading tons of books.
Some he read from cover to cover, some he left unfinished and some he abandoned after reading the contents.

Yet despite of the efforts of the authors he soon realized that without any experience whatsoever reading about testing is like listening how the mountain lake looks like in the morning when You're blind.

He felt he need a change...

User story, ep. 3: Practice

So he started building home from a chimney (as the old Polish saying goes), skipped all the manual testing basics and jumped directly into the Cypress world. He had no idea what he was doing...

Luckily he had learned some JavaScript before and dealt with Visual Studio Code. So after hours of learning he was finally able to set all those colorized expressions into his first test that actually tested something. Nothing fancy, but still...

Mountain weather report

I'm about to set for an expedition to conquer Gerlach summit (The Tatras, Slovakia) in winter. I'd better check the weather report. This is a source code which will check if it is possible on one particular website:
Source code:
                                                                               
describe("when navigate to the mountain weather page on my mobile, viewport: 393 x 851", () => {

        let mountainName = "Gerlach";

    it("should visit the page and make sure it loaded", () => {
        cy.pause()
        cy.visit("https://www.mountain-forecast.com/", {timeout:10000});
        cy.title("include", "Mountain Weather");
        cy.url().should("include", "mountain-forecast")
                .and("eq", "https://www.mountain-forecast.com/")
                .and("contain", "mountain-forecast.com");
    })
    it("should make sure it loaded even more", () => {
        cy.get("img[alt='Mountain-Forecast']").should("be.visible");
        cy.get("a.active").should("exist");
    })
    it("should click on 'Weather maps' button", () => {
        cy.get("a").contains("Weather maps").click({force:true});
        cy.get("div#MM_search").should("be.visible");
    })
    it("should type in mountain name and display forecast table", () => {
        cy.get("#location").as("myFirstAlias")
        cy.get("@myFirstAlias").clear();
        cy.get("@myFirstAlias").type(`${mountainName} {Enter}`, {delay:100});
    })
    it("should scroll down to weather forecast table", () => {
        cy.get('.forecast-table__table-wrapper')
            .scrollIntoView()
            .should("be.visible")
    })
})

User story, ep. 4: More theory

After hours of learning Cypress he realized he needed theory to name somehow what he was doing.

There was one book, the book of books, the testers' bible he inevitably encountered.

The ISTQB syllabus.

A merely 100-pages-volume to encompass the foundations of knowledge every software tester needs. And to prepare for the exam to get a certificate well recognized by future employers.

It's definitely not a thrilling bedtime reading piece of art to be honest. But Lukasz continued reading.
