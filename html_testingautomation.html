Ep. 7: The Cypress breakthrough

User story:
As a person who is eager to become an automation tester I want to be introduced to the best front-end automation tool on the market, so that I can write automation scripts with pure joy.

Acceptance criteria:
1. Download and install Visual Studio Code.
2. Hit 'npm install cypress' in the terminal to install Cypress.
3. Find a Cypress tutorial on Youtube and start writing test scripts.
4. After weeks of heavy decide that You're ready to test some API with another splendid tool.

Scenario: I'm about to set for an expedition to conquer Gerlach summit (The Tatras, Slovakia) in winter. I'd better see the weather report. The test below would check if it is possible on one particular website.

Environment:
Testing framework: Cypress (v. 11.2.0)
OS: Windows 10 Home 64-bit
Browser: Google chrome

Test Results:
- main page loaded:
- page resized to mobile viewport
- redirected to wheather maps subpage
- searchbox working, mountain name found
- wheather forecast table visible

Source code:
                                                                               
describe("when navigate to the mountain weather page on my mobile, 
viewport: 393 x 851", () => {

        let mountainName = "Gerlach";

    it("should visit the page and make sure it loaded", () => {
        cy.pause()
        cy.visit("https://www.mountain-forecast.com/", {timeout:10000});
        cy.title("include", "Mountain Weather");
        cy.url().should("include", "mountain-forecast")
                .and("eq", "https://www.mountain-forecast.com/")
                .and("contain", "mountain-forecast.com");
    })
    it("should make sure it loaded even more", () => {
        cy.get("img[alt='Mountain-Forecast']").should("be.visible");
        cy.get("a.active").should("exist");
    })
    it("should click on 'Weather maps' button", () => {
        cy.get("a").contains("Weather maps").click({force:true});
        cy.get("div#MM_search").should("be.visible");
    })
    it("should type in mountain name and display forecast table", () => {
        cy.get("#location").as("myFirstAlias")
        cy.get("@myFirstAlias").clear();
        cy.get("@myFirstAlias").type(`${mountainName} {Enter}`, {delay:100});
    })
    it("should scroll down to weather forecast table", () => {
        cy.get('.forecast-table__table-wrapper')
            .scrollIntoView()
            .should("be.visible")
    })
})

Scenario: I am about to buy a novel of one of my favourite authors: John Grisham. There is an online bookstore I get in touch with quite often when I need a good stuff to read. The source code below would see if the searchbox of that bookstore website is flawless.

Environment:
Testing framework: Cypress (v. 11.2.0)
OS: Windows 10 Home 64-bit
Browser: Google chrome

Test results:
- main page loaded, searchbox visible
- no search phrase typed in, 'zero result' info provided
- search phrase remains in the searchbox
- search phrase found, results returned
- search phrase not found, info 'zero results' provided

Source code:
                                                                               
describe('when navigate to the online bookstore page', () => {

    let author = "Grisham";
    let dummyPhrase = "sdgrdthfhjgfhkgkgh";
    let searchbox = "#headerSearchInput";
    let website = "https://bonito.pl/";
    let afterHookInfo = "I've just used 2 hooks in my test :)";
  
    beforeEach( () => {
        cy.visit(website);
        cy.url().should("contain", "bonito");
    })

    after( () => {
        cy.log(afterHookInfo);
    })
  
    it("website loaded and searchbox with placeholder visible", () => {
        cy.get(searchbox).should("be.visible");
        cy.get(searchbox).should("have.attr", "placeholder", "Czego szukasz?");
    })
    it("search icon clicked: no redirection", () => {
        cy.get('.menu_ikona')
            .should("be.visible")
            .click();
        cy.url().should("eq", website);
    })
    it("search icon clicked: no-results info provided", () => {
        cy.get(".menu_ikona")
            .should("be.visible")
            .click();
        cy.get(searchbox).should("have.attr", "placeholder", "Wypełnij to pole");
    })
    it("author's name typed in (no Enter), pop up window with hints appears", () => {
        cy.get(searchbox).clear();
        cy.get(searchbox).type(author);
        cy.get("iframe").should("be.visible")
        cy.get(searchbox).should("have.attr", "placeholder", author);
    })
    it("author's name typed in, {Enter}, 24 book items exist, pagination visible", () => {
        cy.get(searchbox).clear();
        cy.get(searchbox).type(`${author} {Enter}`);
        cy.get("h1").contains(author);
        cy.get('.product_card_min_white').should("have.length", 24);
        cy.get("div.H4B").should("be.visible");
    })
    it("dummy phrase typed in, {Enter}, no-results info provided", () => {
        cy.get(searchbox).clear();
        cy.get(searchbox).type(`${dummyPhrase} {Enter}`);
        cy.get('.ps-1 > .T2L').should("have.text", "0 wyników");
    })
  })

Scenario: I would like to login to a banking application. I probably shouldn't try this with a real app. So I found a fake one where people like me can have a go without worrying to much about the consequences.

Environment:
Testing framework: Cypress (v. 11.2.0)
OS: Windows 10 Home 64-bit
Browser: Google chrome

Test results:
- when login with invalid credentials, an error message is displayed
- login with valid credentials possible
- logout possible

Source code:
                                                                                 
    /* FIXTURES Used:
    {
        "username_ID": "username",
        "password_ID": "password"
    }
    */
    
    describe("Login / Logout test", () => {
    
        let invalidUsername = "some username";
        let invalidPassword = "some password";
        const errorMessage = "Login and/or password are wrong.";
        
        before( () => {
            cy.visit("http://zero.webappsecurity.com/index.html")
            // 3 shoulds do wyboru: eq = equal, include, contain
            cy.url().should("include", "index.html")
            cy.url().should("contain", "index.html")
            cy.title().should("equal", "Zero - Personal Banking - Loans - Credit Cards") 
            cy.get("#signin_button").click()
        })
    
    
        it("should try to login with invalid data", () => {
            cy.get("#login_form").should("be.visible")
            cy.get("#user_login").clear().type(invalidUsername, {delay: 100})
            cy.get("#user_password").clear().type(invalidPassword, {delay: 100})
            cy.get("#user_remember_me").check() // optional
            cy.get("input[name='submit']").click()
    
        })
    
        it("should display error message", () => {
            cy.get(".alert-error")
                .should("exist")
                .and("contain", errorMessage)
        })
    
        it("should login to the application", () => {
    
            // fixtures
            cy.fixture("credentials").then(credentials => {
                const username = credentials.username_ID;
                const password = credentials.password_ID; 
    
                // aliases
                cy.get("#user_login").as("loginInput")
                cy.get("@loginInput").clear().type(username, {delay: 100})
                cy.get("#user_password").as("passwordInput")
                cy.get("@passwordInput").clear().type(password, {delay: 100})
    
                cy.get("#user_remember_me").check() // optional
                cy.get("input[name='submit']").click()
            })
    
            cy.get("ul.nav-tabs").should("be.visible")
            
        })
    
        it("should logout from the application", () => {
            cy.contains("username").click()
            cy.get("#logout_link").click()
            cy.url().should("include", "index.html")
        })
    
    })

Ep. 8: The Postman breakthrough

User story:
As a person who is absolutely commited to become an automation tester I want to get familiar with the best API automation tool out there, so that I can write automation scripts with fun and efficiency.

Acceptance criteria:
1. Learn what are API's and what they do with the user input.
2. Download and install Postman.
3. Start using Postman. Don't worry, it has a very friendly GUI.
4. After mastering Postman, start searching for new challenges.

Intro: There are many fake API's to be tested online using Postman. They often provide enough documentation to learn the most common http requests and basic validations of server responses. There is no better way of practice before diving deep into the real API jungle out there.

Environment:
Tested API: https://reqres.in
Testing tool: Postman (v. 10.10.9)
OS: Windows 8.1

Step I:
Read the documentation carefully to see what are the supported HTTP requests and how the respective endpoints should be like.

Step II:
1: Create a collection of most common http requests (GET, POST, PUT, PATCH, DELETE).
2: For every request provide a proper URI (consult the documentation).
3: Use code snippets provided by Postman to validate, e.g. server response code or part of post request body.
4. Send a request body in JSON format to the server. This will be tested when server responds.

Step III:
1: The main bar displays iterations, test duration, total number of tests etc.
2: The lower bar informs how many tests have passed, failed or been skipped.
3: The first test is written on the collection level - it's applied to every request in the collection.
4: Other tests are written separately for each request.

Intro: I worked with the same fake API as in the previous test (above). This time however I went one little step forward and applied global variables to test an HTTP post request.

Environment:
Tested API: https://reqres.in
Testing tool: Postman (v. 10.10.9)
OS: Windows 8.1

Step I:
1: Go to 'Environments' tab, then click 'Globals' to open the panel for setting the variables.
2: Declare 3 variables: for the hostname (host2 = 'reqres.in/api'), for the username (nameGlobalVariable = 'some random name') and for the username occupation (jobGlobalVariable = 'some random job').

Step II:
1: Select POST request. Use the 'host2' variable and insert it into the URL.
2: Click the 'Body' tab, then choose JSON format for the request body to be provided.
3: Create the body of the post request. Use 2 variables previously established for the username and username occupation.

Step III:
1: Create a test to validate the server response.
2: Create a test to validate the 'name' value in the server response body. Don't use hardcoded values. Use previously declared global variable instead.
3: Create a test to validate the 'job' value in the server response body.
4: See the result of the tests you created.
